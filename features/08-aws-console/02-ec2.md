# 08-02: EC2 (Elastic Compute Cloud)

## Overview
Full EC2 management: launch instances (real containers in Floci), lifecycle management, key pairs, security groups, volumes, and elastic IPs.

**boto3 service:** `"ec2"`
**Backend file:** `backend/app/routers/resources/ec2.py`
**Frontend file:** `frontend/app/(dashboard)/[instanceId]/resources/ec2/page.tsx`
**Nav entry:** `{ label: "EC2", href: "ec2", icon: Server }` under Compute group

---

## Backend Tasks

### Instances
- [ ] `GET /instances` ‚Äî `describe_instances` ‚Üí flatten reservations ‚Üí `[{ instance_id, name (tag), state, state_code, instance_type, public_ip, private_ip, key_name, security_groups, subnet_id, vpc_id, availability_zone, launch_time, image_id, iam_instance_profile, platform }]`
- [ ] `POST /instances` ‚Äî `run_instances` body: `{ image_id, instance_type, key_name?, security_group_ids[]?, subnet_id?, min_count=1, max_count=1, user_data_base64?, iam_instance_profile_name?, tags[{Key,Value}]?, block_device_mappings[{device_name, volume_size, volume_type}]? }` ‚Üí `{ instance_id, state }`
- [ ] `POST /instances/{id}/start` ‚Äî `start_instances` ‚Üí `{ previous_state, current_state }`
- [ ] `POST /instances/{id}/stop` ‚Äî `stop_instances` ‚Üí `{ previous_state, current_state }`
- [ ] `POST /instances/{id}/reboot` ‚Äî `reboot_instances` ‚Üí `{ success: true }`
- [ ] `POST /instances/{id}/terminate` ‚Äî `terminate_instances` ‚Üí `{ previous_state, current_state }`
- [ ] `GET /instances/{id}/console` ‚Äî `get_console_output` ‚Üí `{ output: string, timestamp }`
- [ ] `GET /instances/{id}/connect` ‚Äî build SSH command from instance data ‚Üí `{ public_ip, key_name, username (ec2-user/ubuntu based on AMI), ssh_command: "ssh -i {key_name}.pem {user}@{public_ip}" }`

### Key Pairs
- [ ] `GET /key-pairs` ‚Äî `describe_key_pairs` ‚Üí `[{ name, fingerprint, key_pair_id, created_at, tags }]`
- [ ] `POST /key-pairs` ‚Äî `create_key_pair` body: `{ name, key_type?: rsa|ed25519 }` ‚Üí `{ name, key_material (pem), fingerprint }` ‚öÝÔ∏è key_material returned once only
- [ ] `DELETE /key-pairs/{name}` ‚Äî `delete_key_pair`
- [ ] `POST /key-pairs/import` ‚Äî `import_key_pair` body: `{ name, public_key_material: string }` ‚Üí `{ name, fingerprint }`

### Security Groups
- [ ] `GET /security-groups` ‚Äî `describe_security_groups` ‚Üí `[{ id, name, description, vpc_id, ingress_rules[], egress_rules[] }]`
- [ ] `POST /security-groups` ‚Äî `create_security_group` body: `{ name, description, vpc_id? }` ‚Üí `{ id }`
- [ ] `DELETE /security-groups/{id}` ‚Äî `delete_security_group`
- [ ] `POST /security-groups/{id}/ingress` ‚Äî `authorize_security_group_ingress` body: `{ ip_permissions: [{ ip_protocol, from_port, to_port, ip_ranges[{cidr_ip, description}] }] }`
- [ ] `DELETE /security-groups/{id}/ingress` ‚Äî `revoke_security_group_ingress` body: `{ ip_permissions: [...] }`
- [ ] `POST /security-groups/{id}/egress` ‚Äî `authorize_security_group_egress`
- [ ] `DELETE /security-groups/{id}/egress` ‚Äî `revoke_security_group_egress`

### Volumes (EBS)
- [ ] `GET /volumes` ‚Äî `describe_volumes` ‚Üí `[{ id, size, state, volume_type, iops, az, create_time, attachments[{instance_id, device, state}] }]`
- [ ] `POST /volumes` ‚Äî `create_volume` body: `{ size, availability_zone, volume_type?: gp2|gp3|io1|io2|st1|sc1, iops?, snapshot_id? }` ‚Üí `{ id, state }`
- [ ] `POST /volumes/{id}/attach` ‚Äî `attach_volume` body: `{ instance_id, device }` ‚Üí `{ state }`
- [ ] `POST /volumes/{id}/detach` ‚Äî `detach_volume` ‚Üí `{ state }`
- [ ] `DELETE /volumes/{id}` ‚Äî `delete_volume`

### Elastic IPs
- [ ] `GET /elastic-ips` ‚Äî `describe_addresses` ‚Üí `[{ allocation_id, public_ip, association_id, instance_id, domain }]`
- [ ] `POST /elastic-ips` ‚Äî `allocate_address(Domain="vpc")` ‚Üí `{ allocation_id, public_ip }`
- [ ] `POST /elastic-ips/{allocation_id}/associate` ‚Äî `associate_address` body: `{ instance_id }` ‚Üí `{ association_id }`
- [ ] `POST /elastic-ips/{allocation_id}/disassociate` ‚Äî `disassociate_address`
- [ ] `DELETE /elastic-ips/{allocation_id}` ‚Äî `release_address`

### Supporting Lookups
- [ ] `GET /amis` ‚Äî `describe_images(Owners=["self","amazon"], Filters=[{Name:"state",Values:["available"]}], MaxResults=50)` ‚Üí `[{ id, name, description, architecture, platform, creation_date }]`
- [ ] `GET /vpcs` ‚Äî `describe_vpcs` ‚Üí `[{ id, cidr, is_default, state, tags }]`
- [ ] `GET /subnets` ‚Äî `describe_subnets` ‚Üí `[{ id, vpc_id, cidr, az, available_ip_count, is_default }]`

## Frontend Tasks

### Page Structure
- [ ] Create `frontend/app/(dashboard)/[instanceId]/resources/ec2/page.tsx`
- [ ] Implement tabbed layout: Instances | Key Pairs | Security Groups | Volumes | Elastic IPs

### Instances Tab
- [ ] Table columns: Instance ID (mono), Name (tag), State (badge: green=running, yellow=pending, red=stopped/terminated), Type, Public IP, AZ, Key Pair, Launch Time
- [ ] Row actions dropdown: Start / Stop / Reboot / Terminate / Connect Details / Console Output
- [ ] Start/Stop/Terminate require confirmation dialog (with resource ID)
- [ ] "Launch Instance" button ‚Üí multi-step wizard dialog:
  - [ ] Step 1 ‚Äî AMI: dropdown from `GET /amis` (show ID + name + architecture)
  - [ ] Step 2 ‚Äî Instance Type: dropdown (t2.micro, t2.small, t2.medium, t3.micro etc.)
  - [ ] Step 3 ‚Äî Key Pair & Security: key pair dropdown + security group multi-select
  - [ ] Step 4 ‚Äî Network: VPC dropdown ‚Üí Subnet dropdown (filtered by VPC)
  - [ ] Step 5 ‚Äî Storage: EBS volume size + type
  - [ ] Step 6 ‚Äî Advanced: user data textarea (base64 handled automatically), IAM instance profile name, tags (key-value pairs)
  - [ ] Step 7 ‚Äî Review & Launch summary
- [ ] "Connect" action ‚Üí Connection Details dialog: public IP, key pair, pre-formatted SSH command with copy button
- [ ] "Console Output" action ‚Üí modal with console text in monospace scrollable box
- [ ] Auto-refresh state every 5s when any instance is in pending/stopping/shutting-down state

### Key Pairs Tab
- [ ] Table: Name, Fingerprint, Created, Key Pair ID
- [ ] "Create Key Pair" dialog: name input + key type radio (RSA/ED25519) ‚Üí on success show private key in modal with download .pem button (one-time display)
- [ ] "Import Key Pair" dialog: name + public key material textarea
- [ ] Delete button per row (with confirmation)

### Security Groups Tab
- [ ] Table: ID, Name, Description, VPC ID, Inbound rules count, Outbound rules count
- [ ] "Create Security Group" dialog: name, description, VPC dropdown
- [ ] Click row ‚Üí expand inbound/outbound rules table (protocol, port range, source CIDR, description)
- [ ] "Add Inbound Rule" button: protocol dropdown (TCP/UDP/ICMP/All), port range, CIDR input, description
- [ ] "Revoke" button per rule
- [ ] Delete SG button (with confirmation)

### Volumes Tab
- [ ] Table: Volume ID, Size (GiB), Type, State (badge), AZ, Attachment (instance ID + device)
- [ ] "Create Volume" dialog: size, AZ dropdown, volume type, IOPS (for io1/io2)
- [ ] "Attach" dialog per volume: instance ID input + device name
- [ ] "Detach" button per attached volume
- [ ] "Delete" button per unattached volume

### Elastic IPs Tab
- [ ] Table: Public IP, Allocation ID, Associated Instance, Association ID
- [ ] "Allocate Elastic IP" button (no inputs needed)
- [ ] "Associate" dialog: instance ID input
- [ ] "Disassociate" button per associated EIP
- [ ] "Release" button per unassociated EIP
